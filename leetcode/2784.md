# LeetCode 2784 - Check if Array is Good

題目係[呢度](https://leetcode.com/problems/check-if-array-is-good/)

呢條係 Easy 題，sort 一下或者 count frequency 就搞掂。但我好奇：**如果你只可以睇每個 element 一次（streaming），你最少要用幾多 memory？**

O(n) 係顯然的上界。但係有冇可能做到 o(n)？

## 想法

直覺告訴我答案係 Ω(n)——你總要記住睇過咩嘢。但「直覺」唔係證明。

最自然的嘗試係 **information-theoretic argument**：如果個 algorithm 要分辨 `2^k` 個唔同的情況，佢就需要至少 `k` bits。但呢條題的 output 得 yes/no 兩個值，直接睇 output space 只有 1 bit，完全睇唔出 memory 下界。

所以需要另一個工具：**communication complexity**。

## Communication Complexity 背景

設定：Alice 有 input `x`，Bob 有 input `y`，佢哋想一齊計 `f(x, y)`。佢哋可以互相傳 bits，問最少要傳幾多 bits。

經典問題 **Set Disjointness（DISJ）**：Alice 有集合 `A ⊆ [n]`，Bob 有 `B ⊆ [n]`，問 `A ∩ B = ∅` 嗎。Output 同樣只係 yes/no，但可以證明需要 Ω(n) bits。

### Protocol 係一棵 Binary Tree

任何通訊協議都可以畫成一棵 binary tree：每個 internal node 代表「輪到邊個傳 bit」，兩條 edge 代表傳 `0` 或 `1`，每個 leaf 代表一個答案。一個 `c`-bit 協議對應深度 `c` 的 tree，最多 `2^c` 個 leaves。

### Leaf 係 Rectangle

每個 leaf `ℓ` 對應所有走到呢個 leaf 的 `(A, B)` 組合。關鍵觀察：**呢個集合一定係一個 rectangle** `X_ℓ × Y_ℓ`。因為 Alice 傳的 bit 只取決於 `A`，Bob 的只取決於 `B`，走到同一個 leaf 的 inputs 可以獨立替換而唔影響行徑。

所以：**任何 `c`-bit 協議將 input space 劃分成最多 `2^c` 個 monochromatic rectangles**。

### Fooling Set

**Fooling set** 係一堆 YES-instance `{(x₁,y₁), ..., (xₖ,yₖ)}`，滿足：對任意 `i ≠ j`，`(xᵢ, yⱼ)` 或 `(xⱼ, yᵢ)` 係 NO-instance。

**Claim**：任何 monochromatic rectangle 最多包含 fooling set 入面的一個配對。

**證明**：假設 rectangle `R = X × Y` 包含 `(xᵢ, yᵢ)` 同 `(xⱼ, yⱼ)`。因為 `xᵢ ∈ X` 同 `yⱼ ∈ Y`，所以 `(xᵢ, yⱼ) ∈ R`。但 `R` 係 YES-rectangle 而 `(xᵢ, yⱼ)` 係 NO-instance——矛盾。✓

所以需要的 rectangles 數目 ≥ fooling set 大小 `k`，通訊下界係 **Ω(log k)**。

## Lower Bound Proof

定義兩人問題 **GOOD-PARTITION**：Alice 同 Bob 各自持有 array `[1..n]` 中一些 elements（加埋係一個 multiset），問合起嚟係唔係一個 good array。

搵 fooling set：取固定的 good array `[1, 2, ..., n-1, n, n]`，考慮所有將元素分俾 Alice 同 Bob 的方法，共 `2^(n+1)` 個 YES-instance。

**驗證 fooling set 條件**：取兩個唔同的分法 `(A, B)` 同 `(C, D)`，要証 `(A, D)` 係 NO-instance：

- 若 `|A| ≠ |C|`：`|A ∘ D| = |A| + |D| = |A| + (n+1-|C|) ≠ n+1`，長度唔啱，NO。
- 若 `|A| = |C|` 但 `A ≠ C`：`A` 有某 element `x` 係 `C` 冇的，即 `D` 已持有 `x`，`A ∘ D` 入面 `x` 重複出現，NO。

Fooling set 大小 `2^(n+1)`，所以 **GOOD-PARTITION 需要 Ω(n) bits**。

### Reduction 到 Streaming

**Claim**：任何解 GOOD 的 streaming algorithm，都可以用嚟解 GOOD-PARTITION，通訊量等於 streaming 的 space。

**做法**：

1. Alice 將自己的 elements 逐個喂入 streaming algorithm，得到 memory state `s`
2. Alice 將 `s` 傳俾 Bob（`|s|` bits）
3. Bob 從 state `s` 繼續跑，將自己的 elements 喂入，得出答案

呢個係合法的 GOOD-PARTITION 協議，通訊量剛好係 `|s|`。所以：

$$
\text{GOOD streaming space} \geq \text{GOOD-PARTITION communication complexity} = \Omega(n)
$$

## 結論

成條邏輯鏈：

```
GOOD streaming space
  ≥ GOOD-PARTITION communication complexity   （via Alice 傳 memory state）
  ≥ log₂(fooling set size)                    （via rectangle argument）
  = log₂(2^(n+1))
  = Ω(n)
```

呢套工具係分析 streaming algorithms 的通用框架，背後係 Yao (1979) 同 Ablayev (1996) 等人的工作。如果你對呢個方向有興趣，推薦睇 Roughgarden 的 [CS264 notes](http://timroughgarden.org/w16/l/l9.pdf)。
